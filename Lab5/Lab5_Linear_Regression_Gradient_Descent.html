8. Gradient Descent Implementation¶

compute_cost returns the Mean Squared Error (halved, the usual convention for a cleaner gradient). gradient_descent runs batch gradient descent for a fixed number of iterations and keeps track of the cost at every step so we can plot it later.

In [8]:
def compute_cost(X, y, theta):
    m = len(y)
    predictions = X.dot(theta)
    return (1 / (2 * m)) * np.sum((predictions - y) ** 2)


def gradient_descent(X, y, theta, learning_rate, iterations):
    m = len(y)
    cost_history = []

    for i in range(iterations):
        predictions = X.dot(theta)
        error = predictions - y
        gradients = (1 / m) * X.T.dot(error)
        theta = theta - learning_rate * gradients
        cost_history.append(compute_cost(X, y, theta))

    return theta, cost_history

9. Trying Different Learning Rates¶

Before settling on a final model, it helps to see how the learning rate affects convergence. A learning rate that is too small barely moves in 100 iterations, and one that is too large can overshoot the minimum or even diverge.

12. Visualizing the Gradient Descent Path¶

The loss curve above shows cost dropping over time, but it doesn't show how gradient descent moves through parameter space to get there. To see that, we track two of the model's weights (the bias term and the weight for G2, the strongest predictor) at every iteration, then plot them against a 3D surface of the cost function for just those two dimensions (all other weights held fixed at their final trained values). Since MSE is a convex function of the weights, the surface is a bowl shape, and the red path traces the actual route gradient descent took from (0, 0) down toward the minimum.

In [13]:
from mpl_toolkits.mplot3d import Axes3D  # noqa: F401 (registers the 3D projection)

idx_g2 = features.index('G2') + 1  # +1 to account for the bias column at index 0

# rerun gradient descent, this time recording (bias, G2 weight, cost) at every step
theta = np.zeros((n_features, 1))
theta_path = [theta.flatten().copy()]
cost_path = [compute_cost(X_train_b, y_train, theta)]

for i in range(iterations):
    predictions = X_train_b.dot(theta)
    error = predictions - y_train
    gradients = (1 / len(y_train)) * X_train_b.T.dot(error)
    theta = theta - learning_rate * gradients
    theta_path.append(theta.flatten().copy())
    cost_path.append(compute_cost(X_train_b, y_train, theta))

theta_path = np.array(theta_path)
cost_path = np.array(cost_path)

# build a cost surface over the (bias, G2 weight) plane, other weights fixed at their final values
theta0_vals = np.linspace(theta_path[:, 0].min() - 1, theta_path[:, 0].max() + 1, 60)
theta_g2_vals = np.linspace(theta_path[:, idx_g2].min() - 1, theta_path[:, idx_g2].max() + 1, 60)
theta0_grid, theta_g2_grid = np.meshgrid(theta0_vals, theta_g2_vals)

cost_grid = np.zeros_like(theta0_grid)
theta_fixed = theta_final.flatten().copy()

for i in range(theta0_grid.shape[0]):
    for j in range(theta0_grid.shape[1]):
        t = theta_fixed.copy()
        t[0] = theta0_grid[i, j]
        t[idx_g2] = theta_g2_grid[i, j]
        cost_grid[i, j] = compute_cost(X_train_b, y_train, t.reshape(-1, 1))

fig = plt.figure(figsize=(9, 7))
ax = fig.add_subplot(111, projection='3d')
ax.plot_surface(theta0_grid, theta_g2_grid, cost_grid, cmap='viridis', alpha=0.7, edgecolor='none')
ax.plot(theta_path[:, 0], theta_path[:, idx_g2], cost_path, color='red', marker='o', markersize=2, linewidth=1.5)
ax.scatter(theta_path[0, 0], theta_path[0, idx_g2], cost_path[0], color='black', marker='x', s=80, label='start (0, 0)')
ax.set_xlabel('theta (bias)')
ax.set_ylabel('theta (G2 weight)')
ax.set_zlabel('Cost (MSE)')
ax.set_title('Gradient Descent on the Cost Surface')

14. Interpretation¶

The model converges quickly, most of the drop in loss happens in the first hundred iterations and after that it barely changes, so 1000 iterations was more than enough for lr = 0.1. The R2 score of around 0.77 means the model explains a good chunk of the variation in the final grade, though it is far from perfect. That is not surprising since G1 and G2 are almost the same kind of grade as G3, so they end up doing most of the work, while the other features like study time and absences add only a small amount extra. Looking at the scatter plot, most points sit close to the diagonal line, but there are a few students the model clearly gets wrong, most likely the ones whose final grade dropped or jumped compared to their earlier grades for reasons the dataset does not capture (motivation, personal issues, exam difficulty on the day, etc). The gradient descent path plot also shows this clearly, the parameters move in a smooth, direct curve toward the minimum rather than oscillating, which is what we'd expect once the features are scaled and a reasonable learning rate is used. Overall this shows gradient descent can reach essentially the same solution as the closed form / library implementation, as long as the features are scaled and the learning rate is chosen reasonably.
