3.3 Distance Matrix and Decision Boundary Mapping¶

Euclidean Distance: Straight-line distance between two points, calculated as the square root of the sum of squared differences across all features. It is suitable when features are continuous and scaled similarly, and works well for compact, spherical clusters. It is the default and most commonly used metric in KNN.

Manhattan Distance: Sum of absolute differences between coordinates (also called city-block distance). It is suitable for high-dimensional data or when the feature space has a grid-like structure, and is less sensitive to outliers than Euclidean distance since differences are not squared.

To visualize the decision boundary, the data is reduced to 2 dimensions using PCA (since the original data has 30 features and cannot be plotted directly).

In [14]:

def plot_decision_boundary(X_tr, y_tr, k, ax):
    knn = KNeighborsClassifier(n_neighbors=k)
    knn.fit(X_tr, y_tr)

    x_min, x_max = X_tr[:,0].min()-1, X_tr[:,0].max()+1
    y_min, y_max = X_tr[:,1].min()-1, X_tr[:,1].max()+1
    xx, yy = np.meshgrid(np.linspace(x_min, x_max, 200),
                          np.linspace(y_min, y_max, 200))
    Z = knn.predict(np.c_[xx.ravel(), yy.ravel()])
    Z = Z.reshape(xx.shape)

    ax.contourf(xx, yy, Z, alpha=0.3, cmap='coolwarm')
    ax.scatter(X_tr[:,0], X_tr[:,1], c=y_tr, cmap='coolwarm', edgecolor='k', s=20)
    ax.set_title(f'K = {k}')

fig, axes = plt.subplots(2, 2, figsize=(12,10))
for k, ax in zip([1,5,10,20], axes.ravel()):
    plot_decision_boundary(X_pca_train, y_pca_train.values, k, ax)
plt.suptitle('KNN Decision Boundaries (PCA-reduced data)')
plt.tight_layout()
plt.show()

Task 6: Comparative Study with Regression (Lab 3 Integration)¶

In Lab 3 (Linear Regression), the model was evaluated using MAE, MSE, RMSE, and R² Score. These are regression metrics, used when the target is continuous. In this lab, classification metrics (accuracy, precision, recall, F1, ROC-AUC) are used since the target is categorical (Malignant/Benign).

Error-based vs Decision-based Evaluation¶

  • Regression (error-based): Measures how far predicted values are from actual continuous values. Metrics like MAE, MSE, and RMSE quantify the magnitude of prediction error.
  • Classification (decision-based): Measures whether predicted class labels match actual class labels. Metrics like accuracy, precision, recall, and F1 quantify correctness of decisions, not magnitude of error.

R² Score vs Accuracy¶

R² Score explains how much variance in the continuous target is explained by the regression model, ranging typically from 0 to 1 (can be negative). Accuracy is simply the proportion of correctly classified samples. Both give an overall sense of model quality, but R² is about variance explained while accuracy is about correct label counts.

RMSE vs F1 Score¶

RMSE penalizes large errors more heavily since errors are squared before averaging, and is expressed in the same units as the target. F1 Score is the harmonic mean of precision and recall, balancing false positives and false negatives, and does not use magnitude of error at all since classification labels have no numeric distance.

MAE vs Confusion Matrix¶

MAE gives a single average error value across all predictions. The confusion matrix instead breaks predictions into four categories (true positive, true negative, false positive, false negative), giving a much more detailed picture of the types of mistakes a classifier makes, which a single averaged metric like MAE cannot show.

Continuous vs Classification Tasks¶

Regression evaluation assumes an ordered, continuous target where "how far off" is meaningful. Classification evaluation assumes discrete, unordered labels where predictions are either right or wrong, and the cost of different types of mistakes (false positive vs false negative) must be considered separately, especially in domains like medical diagnosis.

Inference¶

Regression metrics measure how far predictions are from actual values, focusing on the magnitude of numerical error (MAE, MSE, RMSE) or variance explained (R²). Classification metrics instead measure whether predicted labels match actual categories, focusing on decision correctness through counts of true/false positives and negatives.

Accuracy alone is insufficient in medical diagnosis because the classes are often imbalanced (as seen here, 357 benign vs 212 malignant), and a model can achieve high accuracy by simply predicting the majority class while still missing many malignant cases. In a healthcare setting, missing a malignant case (false negative) is far more dangerous than a false alarm (false positive), so recall becomes critical since it directly measures how many actual malignant cases were correctly caught. ROC-AUC is also more relevant since it evaluates the model's ability to separate classes across all possible thresholds, not just one fixed cutoff, giving a threshold-independent view of performance.

Overall, regression evaluation is about quantifying error distance on a continuous scale, while classification evaluation is about quantifying correctness of discrete decisions and understanding the trade-off between different types of mistakes. Both frameworks aim to validate model reliability, but the right choice of metric depends entirely on the nature of the prediction task and the real-world cost of different errors.

Task 7: Analytical Questions¶

1. Why is KNN called a lazy learning algorithm? KNN does not learn an explicit model or function during training. It simply stores the training data, and all computation (calculating distances, finding neighbours) happens at prediction time. Since no generalization is done in advance, it is called "lazy" as opposed to "eager" learners like linear regression or decision trees.

2. Why is feature scaling required in KNN? KNN relies on distance calculations between data points. If features are on different scales (e.g. area in hundreds vs smoothness in fractions), features with larger numeric ranges dominate the distance calculation regardless of their actual importance. Scaling ensures all features contribute proportionally.

3. Explain heuristic K selection using the √n rule. A commonly used heuristic sets K equal to the square root of the number of training samples. This gives a reasonable starting point that balances bias and variance without extensive tuning; it is typically rounded to an odd number to avoid tie votes in binary classification.

4. Why is cross-validation more reliable than a single train-test split? A single train-test split gives a performance estimate based on one particular random division of data, which can be biased by which samples happen to fall into the test set. Cross-validation trains and tests the model on multiple different folds and averages the results, giving a more stable and representative estimate of model performance.

5. How does K affect the bias-variance trade-off? Small K values make the model sensitive to individual data points and noise, leading to low bias but high variance (overfitting). Large K values smooth predictions over many neighbours, leading to high bias but low variance (underfitting). An optimal K balances the two.

6. Why is recall more important than accuracy in cancer prediction? In cancer detection, a false negative (predicting benign when the tumour is actually malignant) can delay treatment and be life-threatening. Recall specifically measures how many actual malignant cases were correctly identified, so a high recall reduces the risk of missing dangerous cases, which matters more than overall accuracy in this context.

7. What is the limitation of very large K values? Very large K values consider too many neighbours, including points far from the query point that may belong to a different class. This oversmooths the decision boundary, causing the model to underfit and lose the ability to capture local patterns in the data, reducing classification performance especially near class boundaries.

Conclusion¶

  • The optimal K value was estimated using the heuristic rule (K = √n) and refined using cross-validation, giving a final K value that balances bias and variance.
  • Train-test split ratio (80:20, 70:30, 90:10) caused small variations in accuracy, showing that split choice affects the reliability of performance estimates, especially with smaller test sets.
  • The final KNN model achieved strong performance on accuracy, precision, recall, F1 score, and ROC-AUC, with the confusion matrix and ROC curve giving further insight into class-wise performance.
  • Regression metrics (MAE, MSE, RMSE, R²) evaluate continuous prediction error, while classification metrics (accuracy, precision, recall, F1, ROC-AUC) evaluate discrete decision correctness. Both frameworks serve the same underlying purpose, i.e. validating model reliability, but are suited to fundamentally different types of prediction tasks.
